Monty Hall Eval

Do LLMs reason about expected value, or pattern-match "always switch"?

A multi-turn agentic eval using Inspect AI that tests LLM decision-making through variants of the Monty Hall problem. Two eval variants probe different capabilities:

Standard Monty — Classic switch-or-stay. Both require picking the correct door. Tests if the model discovers the optimal "always switch" strategy.

Generalized Monty — Who Wants to Be a Millionaire mechanics. Stay = guaranteed cash out. Switch = risk it for escalating rewards. Tests risk calibration and EV reasoning.

Key Finding

When staying has higher expected value (scale=1.2: guaranteed $1,000 vs. 2/3 × $1,200 = $800 EV for switching), Opus 4.6 computes EV correctly and cashes out 96% of the time, while GPT-5.4 blindly switches 100% of the time, earning less than the guaranteed payout.

Eval Design

Parameters

ParameterDefaultDescription
n_doors3Number of doors (more doors = more rounds)
n_samples50Games per eval run
framingmoneymoney or lives
final_reward1000Base reward at final round
scale2.0Geometric multiplier between rounds

Metrics

  • switch_rate — fraction of games where model switched at least once
  • mean_switches — average number of switches per game
  • mean_reward — average reward earned
  • win_rate — fraction of games won (generalized only)
  • mean_cashout_round — average round at which game ended (generalized only)

Running

# Generalized with custom params

  -T n_doors=5 -T scale=3.0 -T final_reward=1000 -T framing=lives

Interpretation

The generalized Monty Hall eval reveals a sharp distinction between heuristic reasoning and expected value computation. In the standard problem, both strategies coincide — "always switch" is both the heuristic and the EV-optimal answer. The generalized variant breaks this alignment by making the payoff structure independently configurable.

When scale < 1.5, the guaranteed cash-out has higher EV than the risky switch. A model that truly reasons about expected value will stay. A model that pattern-matches "Monty Hall → switch" will fail.

The high-stakes multi-round variant (5 doors, scale=3) tests a different capability: compounding risk assessment. Even when each individual switch has positive EV, the probability of surviving multiple switches drops rapidly. Neither model tested handles this well yet.
